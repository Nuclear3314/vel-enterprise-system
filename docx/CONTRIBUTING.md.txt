# 貢獻指南

感謝您對 VEL Enterprise System 的興趣！我們歡迎各種形式的貢獻，包括報告問題、提交代碼和改進文檔。

## 報告問題

如果您發現了問題，請通過 GitHub 的 Issues 功能報告。請確保包含以下信息：
- 問題的詳細描述
- 重現問題的步驟
- 預期行為和實際行為
- 環境信息（操作系統、PHP 版本等）

## 提交代碼

1. Fork 本倉庫
2. 創建一個新的分支
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. 提交您的更改
   ```bash
   git commit -m "添加新功能: 描述您的更改"
   ```
4. 推送到您的分支
   ```bash
   git push origin feature/your-feature-name
   ```
5. 創建 Pull Request

## 代碼風格

請確保您的代碼符合以下風格指南：
- 使用 PSR-12 標準格式化 PHP 代碼
- 適當分割代碼，保證每個函數和方法的職責單一
- 添加必要的註釋和文檔

## 測試

請確保您的更改包含適當的單元測試和整合測試。運行測試：
```bash
vendor/bin/phpunit
```

## 文檔

如果您的更改影響了文檔，請一併更新相關文檔。
